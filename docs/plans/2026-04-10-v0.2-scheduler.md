# review-game-core v0.2 — Scheduler Extraction

**Date:** 2026-04-10
**Status:** Implemented
**Scope:** Add a generic concept scheduler extracted from the stats game without pulling in UI text, badge helpers, or game-specific prerequisite policy.

## Goals

1. Provide a reusable scheduler module for concept-centric review games.
2. Keep the API policy-driven so each game can define its own mastery target, spacing, and subskill set.
3. Support persistence-safe rehydration and deterministic next-concept selection.

## Non-goals

- No human-facing scheduler text or state badges.
- No weakest-subskill ranking helpers.
- No hardcoded prerequisite or mixed-concept unlock policy.
- No consumer migration in the same change.

## Public surface

The new root-exported `scheduler` module provides:

- `createSchedulerPolicy()`
- `buildInitialConceptSchedule()`
- `mergeConceptSchedule()`
- `isConceptMastered()`
- `isRetentionDue()`
- `applyConceptOutcome()`
- `applySupplementalConceptExposure()`
- `pickNextConceptId()`

Core state stays concept-centric and serializable. Consumers pass any prerequisite logic through `pickNextConceptId({ isEligible })` instead of the package hardcoding game rules.

## Behavioral rules

- Independent passes advance spacing with `independentGaps`, and reaching the mastery target starts a retention check.
- Supported, assisted, and skipped outcomes keep the concept near the front of the queue by marking `recoveryDue` and shortening the next interval.
- Supplemental exposure updates support state without overwriting the primary outcome history.
- `mergeConceptSchedule()` sanitizes malformed persisted data, preserves valid stored subskill entries, and backfills missing retention timing for already-mastered concepts.
